# 기술 결정 이력 (ADR)

새 결정은 번호를 늘려 아래에 추가. 폐기된 결정은 상태를 "폐기"로 바꾸고 후속 ADR 번호를 적음.

---

## ADR-001: V/L/A를 머신 단위로 분리하지 않는다

**상태**: 채택

**맥락**: Mac=V, Cloud=L, Windows=A 식의 깔끔한 3분할을 검토했으나 leaky함.

**결정**:
- Mac이 V뿐 아니라 grounding/orchestration까지 책임진다
- Windows는 V·L에 관여하지 않는 pure actuator
- Cloud는 추론(L)만, 좌표는 보지 않음

**이유**:
1. 좌표 그라운딩은 비전 결과를 들고 있는 쪽에 살아야 함 → Mac
2. L이 V를 도구처럼 부르는 패턴(hover-후-재관찰)에서 V↔L 왕복이 Mac 내부에 머물러야 비용·지연 폭발 방지
3. 진짜 VLA의 'A'는 모터 토큰 레벨. PyAutoGUI는 그 뒤의 드라이버.

---

## ADR-002: Reflection은 AI 자율 결정 + 시스템 5개 안전망

**상태**: 채택

**결정**: AI가 상태 표현 양식·갱신 시점·기록 내용을 모두 자율 결정. 단 시스템은 다음 5개를 강제한다.

1. **스키마 표류 인지**: 상태 파일 덮어쓸 때 *이전 키 ↔ 새 키 diff*를 다음 프롬프트에 끼움
2. **예측-결과 기록 강제**: reflection 출력에 "예상 / 실제 / 갭" 3필드 필수
3. **세션 종료 강제 consolidation**: `finalize()` 시 자동으로 reflection 프롬프트 진입
4. **콜드 스타트 스캐폴딩**: 첫 세션엔 reflection 장황화 소프트 힌트
5. **토큰 예산**: state 파일은 N 토큰 이하 유지, 압축 책임은 AI에

**원칙**: 시스템은 *트리거*만, AI는 *내용*.

## ADR-003: 액션 모델 = D (full tool-use loop)

**상태**: 채택

**결정**: LLM이 도구(`mouse_move`, `click`, `drag`, `get_screenshot` 등)를 직접 호출하고, `end_turn`을 부를 때까지 반복.

**제약**: 턴당 도구 호출 예산 = 8회 (초기값). 초과 시 시스템이 강제로 `end_turn` 처리 후 다음 사이클로.

**이유**:
- "hover-툴팁-판단-클릭"을 한 턴 내에 처리 가능
- 비용은 LLM이 도구를 활발히 쓸 때만 증가. 적게 쓰면 단일 액션 안과 비슷.

**대안 검토**: A(단일 액션+JSON), B(A+턴내 vision tool), C(액션 시퀀스). D가 hover-재관찰 패턴을 가장 자연스럽게 표현하므로 채택.

---

## ADR-004: 매 턴 stateless, 모든 기억은 파일

**상태**: 채택

**결정**: LLM 호출 간 롤링 히스토리 없음. 이전 턴 대화는 다음 호출에 안 들어감. AI의 모든 기억은 `meta_notes` / `strategy_note` / `state.{ext}` 파일.

**이유**:
- Voyager / MemGPT 패턴
- 컨텍스트 폭발 방지
- "중요하면 파일에 적어둬라"를 AI에게 명시적으로 강제

---

## ADR-005: 좌표는 픽셀 절대값

**상태**: 채택

**결정**: 정규화 좌표가 아닌 픽셀 `[x, y]` 사용.

**이유**: 고정 창모드이므로 절대좌표가 단순. LLM 좌표 추론 정밀도도 절대값이 안정적.

---

## ADR-006: 모든 액션 후 2초 대기 (Windows 내부)

**상태**: 채택

**결정**: Windows 액션 함수는 *내부에서* 2초 대기 후 반환. 호출자(Mac)는 별도 sleep 불필요.

**이유**:
- 애니메이션·전환·툴팁 렌더링 시간 확보
- 매 액션 일관된 settle time
- 호출자가 매번 sleep 관리할 필요 없음

**경계**: 비전 도구(`get_screenshot`, `get_crop_image`)는 대기 없음.

**제약**: 2초 안에 의미 있게 진행 가능한 게임만 후보로 선정.

---

## ADR-007: 호버 함수 제거, mouse_move + 자동 대기로 합성

**상태**: 채택

**결정**: 별도 `hover(x, y, duration)` API 없음. `mouse_move(x, y)` + ADR-006 2초 대기 = 사실상 호버.

**이유**: 인터페이스 단순화. duration이 고정이라 파라미터 의미 없음.

**고지**: base system prompt에 "마우스 이동 후 2초 내 뜨는 툴팁은 다음 스크린샷에 잡힘"을 명시 (시스템 invariant).

---

## ADR-008: 동기 단방향, 비동기 호환성 미고려

**상태**: 채택

**결정**: Mac↔Windows는 동기 HTTP 요청/응답. 비동기는 PoC 성공 후 별도 검토. 호환성을 위한 추상화는 도입하지 않음.

**이유**: PoC 단순화.

---

## ADR-009: 에러 처리 분리 — transient는 시스템, 의미적은 LLM

**상태**: 채택

**결정**:
- **시스템 레벨 transient** (네트워크 블립, 캡처 실패, 일시적 포커스 손실): Windows/Mac 내부 자동 재시도 (최대 3회). LLM에 안 알림.
- **의미적 에러** (좌표 화면 밖, 클릭 후 화면 무변화 등): 다음 턴 state로 보고. LLM이 판단.
- **3회 retry 한계 초과**: 시스템 에러로 LLM에 보고.

---

## ADR-010: 단일 레포 + 4 폴더 (controller / executor / shared / games)

**상태**: 채택

**결정**: Mac/Windows 코드를 한 레포에 폴더로 분리. 공유 스키마(`shared/schema.py`)를 두 쪽에서 import.

**이유**: PoC 단계에 다중 레포 오버헤드 불필요.

---

## ADR-011: 시스템 invariant는 controller 코드의 base system prompt에 둔다

**상태**: 채택

**결정**: "모든 액션 후 2초 대기" 같은 시스템 계약은 `controller/prompts/` 안 코드 상수로 두고 base system prompt에 합친다. `_shared/meta_notes.md` (AI 갱신 가능 메모리)와는 분리.

**이유**: 시스템 계약과 AI 기억의 권한·출처를 섞지 않기 위함.
